1- Go to source folder
cd ./experiments/jaguar-2015/vraptor-40028d8/vraptor-core
USE 'vraptor-core' as your project!!!!!

2- Open eclipse, File-> Import as -> Existing Maven Project (need to get maven and maven eclipse plug-in (m2e) installed before).
Import the vraptor-core module.
Do not worry about an error in pom.xml saying "Plugin execution not covered by lifecycle configuration".

3- Make sure you are using Java 1.7 to compile it.
Select vraptor project.
Right-click, then go to Properties.
Click on Java Compiler.
Compiler compliance lever = 1.7
Apply.

4- Select src/test/java folder and then Rigth click on it -> Run as -> JUnit test.
Only this test should fail: br.com.caelum.vraptor.deserialization.XStreamXmlDeserializerTest.shouldBeAbleToDeserializeAPersonWithDog

5- Select src/test/java folder and then Rigth click on it -> Run as -> Run Configurations.
On the left side, double click Jaguar Fault Localization.
On the first tab (Jaguar), chose control-flow or data-flow.
On the first tab (Jaguar), add this in the Includes field: br.com.caelum.vraptor*
On the second tab (Test), make sure you have the option "Run all tests in the selected project, package or source folder" and then src/test/java from vraptor.
On the third tab (Main) you should be using jdk1.7.0_75 (we have experienced erros with other versions).

6- Run and whatch.

The error is at:
br.com.caelum.vraptor.core.MethodInfo 67
