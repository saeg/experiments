1- Go to source folder
cd ./experiments/jaguar-2015/commons-math-CRVG_AK_1/source

2- Open eclipse, File-> Import as -> Existing Maven Project (need to get maven and maven eclipse plug-in (m2e) installed before).
Do not worry about an error in pom.xml saying "Plugin execution not covered by lifecycle configuration".

3- Select src/test/java folder and then Rigth click on it -> Run as -> JUnit test.
Only this test should fail: org.apache.commons.math.random.CorrelatedRandomVectorGeneratorTest.testMath226

4- Make sure you are using Java 1.7 to compile it.
Select commons-math project.
Right-click, then go to Properties.
Click on Java Compiler.
Compiler compliance lever = 1.7
Apply.

5- Select src/test/java folder and then Rigth click on it -> Run as -> Run Configurations.
On the left side, double click Jaguar Fault Localization.
On the first tab (Jaguar), chose control-flow or data-flow.
On the second tab (Test), make sure you have the option "Run all tests in the selected project, package or source folder" and then src/test/java from commons-math.
On the third tab (Main) you should be using jdk1.7.0_75 (we have experienced erros with other versions).

6- Run and whatch.

The error is at:
org.apache.commons.math.random.CorrelatedRandomVectorGenerator 259
